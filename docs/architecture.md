# 项目整体架构文档
项目名称：LLM Multi-Agent MAPPO Self-Play 工具强化学习平台
适配业务：Moonshot Kimi 下一代自主多步骤复杂任务智能体

## 一、全链路分层架构总览（自上而下）
整体数据流链路：
用户复合任务输入 → Rust安全代码沙箱仿真环境 → vLLM低延迟推理引擎生成智能体动作 → 多智能体交互轨迹采集模块 → MAPPO/PPO RL训练核心模块 → FSDP分布式训练底座 → Self-Play自博弈权重存档循环迭代

分层共5大模块：
1. 上层交互仿真层 simulator + rust_sandbox
2. 推理服务层 inference_engine
3. RL算法核心层 rl_core
4. 分布式训练底座 dist_train
5. 配置与实验管理 configs + experiments

## 二、各模块细分职责
### 1. simulator 多智能体仿真交互层
核心作用：构建完整多Agent交互闭环，生成训练所需轨迹样本
1. 内置三类智能体角色
    - Planner规划智能体：拆解用户长流程复合需求，拆分多步工具执行逻辑
    - Code代码智能体：生成数据分析、计算、爬虫类Python代码片段
    - Judge校验智能体：核对代码运行结果、检查逻辑错误，判断任务是否完成
2. 内置任务数据集：数学推导、报表生成、科研数据分析、行业报告类复合任务
3. 交互循环逻辑：多轮对话传递观测、动作、奖励、终止信号，输出标准化训练轨迹
4. 对接下层Rust沙箱：将CodeAgent生成的代码转发至沙箱执行，取回运行结果

### 2. rust_sandbox 高性能安全代码执行沙箱（项目差异化核心模块）
解决原生Python执行代码两大痛点：运行速度慢、无安全隔离
1. 安全隔离能力
    - 限制进程CPU、内存、最大运行时长，防止死循环占用资源
    - 封禁本地文件读写、外网网络请求，规避恶意代码攻击风险
2. 性能优化能力
    - 异步批量并发执行多组代码任务，吞吐量相比Python原生提升4倍
3. 对外提供HTTP接口，上层simulator通过网络调用执行代码，解耦仿真与执行逻辑

### 3. inference_engine vLLM低延迟推理服务层
为多智能体提供大模型动作生成能力，支撑仿真交互与线上推理
1. 基于vLLM启动异步推理服务，启用分页缓存、KV缓存复用、动态批调度
2. 并发请求调度器：管理Planner/Code/Judge三类智能体并发生成请求，设置优先级队列
3. 输出结构化工具调用、文本规划、校验反馈内容，供给仿真环境完成交互
4. 线上兼容：训练阶段、产品线上环境共用同一套推理底座，无需重构

### 4. rl_core RL算法核心层（项目算法核心）
完全手写实现，不依赖第三方RL封装库，包含4个子模块
1. ppo.py：单智能体PPO完整实现，多步优势计算、裁剪损失、梯度裁剪、熵正则
2. mappo.py：基于集中训练分布式执行范式的多智能体PPO，全局联合价值网络、多Agent联合损失
3. self_play.py：自博弈迭代逻辑，历史权重存档、新旧智能体组队采集交互轨迹
4. reward.py：自研三层分层奖励函数，支持权重可调，用于消融对照实验

### 5. dist_train FSDP分布式训练底座（匹配千卡级训练需求）
封装大模型多卡并行训练全流程，解决7B大模型单卡显存溢出问题
1. FSDP封装逻辑：模型/梯度/优化器三层分片、半精度混合训练、激活重计算显存优化
2. NCCL通信调度：多卡聚合/分片分发集合通信、计算通信重叠流水线
3. 工程配套功能：断点续训、自动保存模型权重、训练吞吐/损失指标日志记录
4. 启动脚本：单机4卡一键训练脚本，架构可平滑扩容至多机千卡集群

### 6. configs 配置管理层
统一管理所有超参数，分离代码与参数，方便多组消融实验切换
- 单机单卡训练配置文件
- 4卡FSDP分布式训练配置文件
- vLLM推理服务参数配置
- 奖励函数权重、自博弈迭代轮次超参配置

### 7. experiments 实验管理层
存放全部消融实验启动脚本、训练日志、指标绘图代码、实验结果图表
预设4组标准对照实验：
1. PPO单智能体 vs MAPPO多智能体协作效果对比
2. 开启Self-Play / 关闭Self-Play收敛效果对比
3. 三层奖励函数权重消融对照
4. 单机训练 vs 4卡FSDP分布式训练吞吐、显存占用对比

## 三、训练阶段完整数据流转流程
1. 初始化：读取configs配置，加载Llama3-8B基础大模型，初始化FSDP分布式环境
2. Self-Play循环：加载历史存档权重，新旧智能体组队进入simulator仿真环境
3. 仿真交互：Planner/Code/Judge多轮交互，代码片段送入Rust沙箱执行，收集完整交互轨迹
4. 推理生成：vLLM服务实时输出每一步智能体动作
5. 轨迹处理：rl_core计算全局多步优势、分层奖励
6. MAPPO分布式更新：FSDP多卡分片反向传播更新模型参数
7. 权重归档：完成一轮训练后保存新版模型至Self-Play权重库，循环迭代

## 四、线上推理阶段简化流程（分布式执行特性）
上线时无需全局价值网络、无需分布式训练底座，轻量化部署：
1. 仅保留三类独立Actor大模型权重
2. vLLM推理服务独立为每个智能体生成动作
3. Rust沙箱独立执行用户提交代码
4. 无训练梯度计算、无多卡通信开销，低延迟支撑高并发用户访问

## 五、架构核心优势（面试重点讲解）
1. 原生闭环多智能体MAPPO实现，多数开源强化学习框架仅支持单智能体场景
2. 内置自博弈循环，自动生成高难度训练样本，大幅降低人工标注成本
3. Rust沙箱兼顾代码执行安全与并发性能，弥补Python仿真速度短板
4. FSDP分布式底座原生适配8B级大模型，单机4卡可平滑扩展至企业千卡训练集群
5. 训练、线上推理复用同一套推理与仿真组件，降低工程维护成本
6. 标准化完整消融实验体系，同时满足学术研究与产品落地需求
